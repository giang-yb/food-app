# 1. Context

**Problem:**
Users need an appetizing, fast, and convenient entry point to the food delivery platform. The home page must immediately engage users with vibrant visuals, clear promotions, and easy navigation to food categories and products.

**Affected Modules:**
- `features/home` ‚Äî Home page with all sections (hero, categories, products, promo)
- `shared/components` ‚Äî Reusable UI components (button, card, rating-stars, quantity-selector)
- `layout` ‚Äî Header/footer integration via main-layout

**Non-Goals:**
- Cart functionality (Phase 4)
- User authentication (Phase 8)
- Real backend API integration (mock data only)
- Search functionality (Phase 6)
- Admin panel (Phase 9)

---

# 2. Constraints

**Language:** TypeScript (Angular 19)
**Architecture:** Angular Standalone Components with Signals

**Rules:**
- Do NOT modify unrelated modules
- Do NOT introduce new external dependencies (CSS-only animations)
- Use existing shared components from `shared/components/ui-kit/`
- All components must be standalone (no NgModule dependency)

**Performance Budgets:**
- Initial load: < 2s on 3G
- Carousel transition: 500ms max
- Component render: < 100ms per section

---

# 3. Conventions

> Shared rules ALL tasks must follow.

**Naming:**
- Components: `PascalCase` ‚Äî e.g. `HeroBannerComponent`, `CategoryCardComponent`
- Files: `kebab-case` ‚Äî e.g. `hero-banner.component.ts`, `home.service.ts`
- Variables/methods: `camelCase` ‚Äî e.g. `activeBannerIndex`, `getCategories()`
- Constants: `UPPER_SNAKE_CASE` ‚Äî e.g. `AUTO_PLAY_INTERVAL`

**Angular Patterns:**
- Standalone components (no NgModule)
- Signals for reactive state (`signal()`, `computed()`)
- OnPush change detection strategy
- Lazy loading for feature routes

**Error Handling:**
- Never return null ‚Äî use empty arrays/objects instead
- Catch errors in services, emit via signal
- Fallback UI states for every async operation

**Logging:**
- Console logging for development (remove in production)
- Format: `[HomePage] action={description}`
- Required at: component init, async operations, errors

**Testing:**
- Framework: Jasmine + Karma (Angular default)
- Naming: `should_<expected>_when_<condition>`
- Coverage: >= 80% on new code

**Task Size:**
- Max 200 LOC per task (excluding tests)
- If exceeds, split into subtasks

```

  id: string           ‚Äî unique identifier
  name: string         ‚Äî product name

  imageUrl: string     ‚Äî product image path

  id: string           ‚Äî unique identifier
  endTime: Date        ‚Äî sale end timestamp
  products: Product[]  ‚Äî featured sale products
}
```

## HomeStore Interface

```
HomeStore {
  // State signals
  banners: Signal<Banner[]>
  activeBannerIndex: Signal<number>
  categories: Signal<Category[]>
  featuredProducts: Signal<Product[]>
  flashSale: Signal<FlashSale | null>
  countdownSeconds: Signal<number>
  isLoading: Signal<boolean>
  error: Signal<string | null>

  // Computed
  countdownDisplay: Computed<string> // "HH:MM:SS" format

  // Actions
  setActiveBanner(index: number): void
  nextBanner(): void
  startCountdown(): void
  stopCountdown(): void
  loadAll(): void
}
```

## HomeService Interface

```
HomeService {
  getPromotionalBanners(): Observable<Banner[]>
  getCategories(): Observable<Category[]>
  getFeaturedProducts(): Observable<Product[]>
  getFlashSale(): Observable<FlashSale | null>
}
```

# 5. Target Architecture

**Components:**
- `HomePageComponent` ‚Äî Main orchestrator, loads all data on init
- `HeroBannerComponent` ‚Äî Carousel with auto-play and manual navigation
- `CategoriesShowcaseComponent` ‚Äî Horizontal scrollable category cards
- `FeaturedProductsComponent` ‚Äî Product grid with hover effects
- `PromotionalSectionComponent` ‚Äî Flash sale banner with countdown timer
- `CountdownTimerComponent` ‚Äî Reusable countdown display (HH:MM:SS)
- `ProductCardComponent` ‚Äî Individual product display card

**Data Flow:**

```
HomePageComponent (init)
  ‚Üí HomeStore.loadAll()
    ‚Üí HomeService.getPromotionalBanners() ‚Üí banners signal
    ‚Üí HomeService.getCategories() ‚Üí categories signal
    ‚Üí HomeService.getFeaturedProducts() ‚Üí featuredProducts signal
    ‚Üí HomeService.getFlashSale() ‚Üí flashSale signal + countdown start

HeroBannerComponent
  ‚Üí reads banners signal from store
  ‚Üí auto-play via interval timer (5s)
  ‚Üí manual nav via setActiveBanner()

CategoriesShowcaseComponent
  ‚Üí reads categories signal from store
  ‚Üí click ‚Üí navigate to /category/:slug

FeaturedProductsComponent
  ‚Üí reads featuredProducts signal from store
  ‚Üí click product ‚Üí navigate to /product/:id
  ‚Üí click "Add to Cart" ‚Üí placeholder (Phase 4)

PromotionalSectionComponent
  ‚Üí reads flashSale signal + countdownSeconds
  ‚Üí countdown display updates every second
```

**Key Decisions:**
1. Store-based state ‚Äî signals for reactive updates, no RxJS subscriptions in components
2. Service layer returns Observables ‚Äî converted to signals in store
3. Mock data in services ‚Äî easy swap to real API later
4. Countdown timer is standalone ‚Äî can be reused in flash sale page (Phase 6)
